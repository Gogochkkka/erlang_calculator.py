Моносервисная модель Эрланга с потерями (Erlang B)

Назначение и область применения

Модель Эрланга применяется для оценки объёма ресурса при обслуживании заявок на передачу трафика сервисов реального времени (речь, видеоконтент). Используемая шкала времени — моменты поступления заявок. Модель носит локальный характер и обычно относится к фрагменту сети: соединительная линия, приёмопередающее оборудование отдельной соты, операторы call-центра.

Входной поток предполагается пуассоновским. Это допущение основано на том, что число пользователей услуг связи достаточно велико, заявки поступают независимо друг от друга с малой интенсивностью, а суммарная интенсивность стремится к константе. Интервалы времени между последовательными поступлениями заявок в этом случае имеют экспоненциальное распределение.

Функциональные компоненты модели

  • Ресурс передачи информации — выражается целым числом виртуальных каналов v. За единицу ресурса может приниматься битовая скорость виртуального канала, тайм-слот, оператор.
  • Допуск заявок — для обслуживания используется любая свободная единица ресурса, все они идентичны с точки зрения качества сервиса. Если весь ресурс занят, заявка получает отказ и не возобновляется.
  • Алгоритм управления передачей — выделенный ресурс не меняется за время предоставления сервиса (свойство сервисов реального времени).

Математическое описание

Время обслуживания заявки предполагается экспоненциально распределённым с параметром μ (среднее время обслуживания h = 1/μ). Функционирование модели описывается марковским процессом. Состояние модели задаётся числом занятых каналов i (i = 0, 1, …, v). Динамика изменения состояний определяется двумя типами событий: поступление новой заявки (интенсивность λ) и завершение обслуживания одной из i заявок (интенсивность i·μ).

Из системы уравнений равновесия выводятся соотношения детального баланса, которые позволяют получить стационарные вероятности состояний:

P(i) = P(0) · (ai / i!),    i = 0, 1, …, v,
где a = λ / μ — интенсивность предложенной нагрузки.

Потенциальная загрузка ресурса (эрланги)

Интенсивность предложенного трафика a измеряется в эрлангах (Эрл) и численно равна среднему числу заявок, поступающих за среднее время обслуживания одной заявки: a = λ · h. По определению, a представляет собой среднее число потенциальных соединений в системе с неограниченным ресурсом (v = ∞). Эта характеристика не зависит от качества работы системы и служит мерой поступающей нагрузки.

Формула Эрланга B

Доля потерянных заявок (вероятность блокировки) совпадает с долей времени занятости всех v каналов (свойство PASTA) и обозначается E(v, a):

E(v, a) = ( av / v! ) / ( Σi=0v ai / i! )

Для практических расчётов используется рекурсивный алгоритм:
E(0, a) = 1;   E(r, a) = (a · E(r−1, a)) / (r + a · E(r−1, a)),   r = 1..v.

Основные рассчитываемые характеристики

  • Вероятность блокировки E(v, a) — доля потерянных заявок.
  • Среднее число занятых каналов m = a · (1 − E(v, a)).
  • Коэффициент использования канала δ = m / v.

Свойства формулы Эрланга

  • Устойчивость: формула Эрланга сохраняет свой вид для произвольного распределения времени обслуживания заявки, если его среднее зафиксировано.
  • Монотонность: вероятность блокировки E(v, a) возрастает с ростом a и убывает с ростом v.

Прямые и обратные задачи

Калькулятор решает три типа задач:

  • Прямая задача: по v и a определить E и m.
  • Обратная задача 1: по a и допустимой E найти минимальное v.
  • Обратная задача 2: по v и допустимой E найти максимальную нагрузку a.
